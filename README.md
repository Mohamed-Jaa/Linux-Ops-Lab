## üß† Learning Methodology
This project follows a **"Problem-Based Learning"** approach. Instead of following random tutorials, I treat every concept as a **Ticket (Task)** that needs to be solved:

1.  **The Scenario:** A simulation of a real infrastructure problem (e.g., "Server crashed due to full disk").
2.  **The Investigation:** diagnosing the issue using Linux analysis tools.
3.  **The Solution:** Writing a script or configuration to fix the issue permanently.
4.  **The Documentation:** explaining the "Why" and "How" for future reference.
